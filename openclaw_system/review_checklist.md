# Review Checklist

## Проверить по пунктам

1. **Соответствие задаче**
   - результат закрывает исходную задачу
   - нет ухода в сторону

2. **Соответствие плану**
   - шаги выполнены по согласованной логике
   - нет самовольного обхода плана

3. **Качество изменений**
   - изменения выглядят целевыми
   - нет явного мусора
   - нет очевидных лишних правок

4. **Проверка Tester**
   - тест действительно был
   - результат PASS понятен и подтверждён
   - если тест слабый или неполный, review должен вернуть FIX

5. **Решение**
   - если всё нормально → OK
   - если есть проблемы → FIX
