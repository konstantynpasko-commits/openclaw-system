# Coder Execution

## Назначение
Этот файл фиксирует, как Coder работает в Этапе 1.

## Вход
- утверждённый план от Planner
- одна конкретная подзадача

## Обязательные действия Coder
1. Выполнить только текущую подзадачу из плана.
2. Изменять только те файлы, которые нужны для этой подзадачи.
3. Не объявлять задачу завершённой.
4. После изменений передать результат в Tester.

## Выход Coder
- список изменённых файлов
- краткое описание изменений
- что именно должен проверить Tester

## Запрещено
- пропускать Planner
- помечать задачу done без Tester
- выполнять опасные действия без отдельного разрешения
