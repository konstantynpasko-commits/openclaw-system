# Scenario Selection Rules

## Bugfix
Использовать когда:
- есть ошибка
- есть поломка поведения
- нужен минимальный исправляющий change

## Feature
Использовать когда:
- добавляется новая возможность
- есть новый пользовательский или системный результат

## Refactor
Использовать когда:
- поведение должно сохраниться
- меняется структура, читаемость, поддерживаемость

## Правило
Перед началом implementation задача должна быть отнесена к одному из шаблонов.
