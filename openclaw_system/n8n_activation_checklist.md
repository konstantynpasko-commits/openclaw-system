# n8n Activation Checklist

## Что нужно для перехода n8n из planned в active
1. Команда `n8n` должна быть доступна, либо должен быть подтверждён другой валидный способ запуска.
2. Должен быть понятен способ хранения конфигурации n8n.
3. Должен быть подтверждён каталог данных или способ persistence.
4. Должен быть выполнен минимальный smoke-check запуска.
5. Результат должен быть зафиксирован в integrations_state.md.

## До выполнения этого checklist
- n8n считается архитектурно запланированным, но не подтверждённо рабочим
