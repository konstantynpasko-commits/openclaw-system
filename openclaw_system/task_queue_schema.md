# Task Queue Schema

## Назначение
Этот файл описывает task queue слоя системы OpenClaw для Этапа 4.

## Статусы задач
Используются только следующие статусы:
- `pending`
- `running`
- `failed`
- `done`

## Основные файлы
- `task_queue.json` — основной список задач очереди
- `task_queue_rules.md` — правила переходов и обработки
- `task_queue_state.md` — текущее состояние очереди

## Минимальная структура записи задачи
Каждая задача должна содержать:
- `id`
- `title`
- `status`
- `project_id`
- `created_at`
- `updated_at`
- `owner`
- `stage`
- `notes`

## Правила переходов
- новая задача создаётся как `pending`
- когда задача взята в работу, статус меняется на `running`
- если выполнение провалено, статус меняется на `failed`
- если задача завершена успешно, статус меняется на `done`

## Жёсткое правило
- любые новые системные задачи в рамках новой архитектуры должны ориентироваться на эти статусы очереди
