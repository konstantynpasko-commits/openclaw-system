# Stage 1 Execution Cycle

## Цель этапа
Ввести обязательный базовый цикл:

Planner → Coder → Tester

## Порядок работы
1. **Orchestrator** принимает задачу.
2. **Planner** оформляет план по шаблону `planner_template.md`.
3. **Coder** выполняет только утверждённую подзадачу по правилам `coder_execution.md`.
4. **Tester** проводит минимальную проверку по `tester_checklist.md`.
5. Если Tester вернул FAIL — задача возвращается в Coder.
6. Если Tester вернул PASS — результат можно считать прошедшим Этап 1.

## Ограничение Этапа 1
- Reviewer пока не является обязательным отдельным системным слоем.
- Но тест обязателен уже на этом этапе.

## Правило фиксации
Для каждой значимой задачи должны быть явно видны:
- план
- изменения
- проверка
