# Project Memory Schema

## Назначение
Этот файл описывает структурированный memory-слой системы OpenClaw внутри папки openclaw_system.

## Основные файлы памяти

### 1. memory_index.md
Содержит карту memory-слоя:
- какие memory-файлы существуют
- за что отвечает каждый файл
- где искать нужный тип информации

### 2. system_state.md
Хранит текущее состояние системы:
- какой этап реализован
- какие контуры уже действуют
- что сейчас считается рабочим стандартом

### 3. progress_log.md
Хранит последовательность значимых изменений:
- что было сделано
- когда
- каким этапом это считается

### 4. active_context.md
Хранит активный рабочий контекст:
- текущий этап
- текущая задача
- следующий ожидаемый шаг

### 5. decisions_log.md
Хранит архитектурные и операционные решения:
- что решили
- почему решили
- что это меняет

## Правила использования
- после значимого изменения обновлять минимум `system_state.md` и `progress_log.md`
- при смене активного блока обновлять `active_context.md`
- архитектурные решения записывать в `decisions_log.md`
- `memory_index.md` должен оставаться точкой входа в memory-слой
