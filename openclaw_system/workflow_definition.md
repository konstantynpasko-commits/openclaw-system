# Workflow Definition

## Целевой полный цикл

task
→ plan
→ code
→ test
→ review
→ если ошибка → возврат к coder
→ если ok → следующая задача

## Реализация на текущем Этапе 2

task
→ plan
→ code
→ test
→ если FAIL → возврат к coder
→ если PASS → review
→ если FIX → возврат к coder
→ если OK → задача считается завершённой

## Правило завершения
- ни одна задача не считается завершённой без теста и review
- после любого существенного исправления задача обязана снова пройти test и review
- финальная допустимая последовательность завершения: PASS → OK
