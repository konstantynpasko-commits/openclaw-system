# GitHub Workflow Rules

## Основной принцип
GitHub слой должен поддерживать workflow системы, а не жить отдельно от него.

## Правила
1. Изменения должны быть связаны с конкретной задачей или этапом.
2. Перед подготовкой коммита должно быть понятно:
   - что меняется
   - зачем меняется
   - как это проверялось
3. Для coding-задач ориентир остаётся тем же:
   - plan
   - code
   - test
   - review
4. GitHub Actions используется как дополнительный автоматический контроль.
5. Нельзя считать GitHub Actions единственной проверкой качества.

## Минимальный контур
- рабочие файлы изменены
- локальная проверка выполнена
- результат зафиксирован
- затем изменения могут идти в GitHub workflow
