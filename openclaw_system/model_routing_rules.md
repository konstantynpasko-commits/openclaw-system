# Model Routing Rules

## Основной маршрут
- по умолчанию основной coder/runtime path остаётся за `codex`

## Вспомогательные модели
- `pi`, `opencode`, `gemini`, `kimi` рассматриваются как вспомогательный слой
- их можно использовать только как осознанное расширение, а не как автоматическую замену основного контура

## Жёсткие правила
1. Не переключать систему на вспомогательную модель без явной причины.
2. Не считать allowlisted agent автоматически стабильным.
3. Не смешивать несколько вспомогательных моделей в одном простом рабочем цикле без необходимости.
4. Основной архитектурный маршрут должен оставаться предсказуемым.
