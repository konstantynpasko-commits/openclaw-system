# Runtime Exec Rules

## Правила выполнения команд
1. Перед важными runtime-действиями нужно понимать, доступна ли команда через PATH.
2. Если команда не найдена, нельзя считать это логической ошибкой задачи — это runtime limitation.
3. Если известен абсолютный путь к инструменту, использовать его явно.
4. Для shell-команд, влияющих на систему, сохраняется правило осторожности и явного разрешения на опасные действия.

## Правила интерпретации ошибок
- `command not found` = проблема runtime/PATH, а не автоматически проблема логики задачи
- успешный запуск команды = подтверждение доступности инструмента
- отсутствие бинаря должно фиксироваться в runtime state

## Практическое правило
Для нестабильного runtime сначала проверять доступность команды, потом строить действие вокруг неё.
