# Guardrails Layer

## Назначение
Этот файл фиксирует guardrails как отдельный системный слой.

## Цель guardrails
- снижать риск опасных действий
- не путать быстрые полезные действия с действиями, которые требуют подтверждения
- удерживать систему в предсказуемом режиме

## Базовый принцип
Любое действие должно попадать в один из режимов:
- safe
- caution
- restricted
- blocked

## Правило
Чем выше риск действия, тем выше требование к проверке, подтверждению и ограничениям.
