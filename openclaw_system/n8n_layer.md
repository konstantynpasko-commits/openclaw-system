# n8n Layer

## Назначение
Этот файл фиксирует n8n как целевой side-automation слой системы.

## Роль n8n
- внешние интеграции
- маршрутизация событий
- автоматизация side-workflows
- связка между OpenClaw и внешними сервисами, где это уместно

## Правильное место n8n в архитектуре
- n8n не должен быть мозгом системы
- n8n не заменяет Planner, Coder, Tester, Reviewer или Orchestrator
- n8n используется как интеграционный и event/workflow слой

## Честный статус на текущем этапе
- слой зафиксирован как часть архитектуры
- активная установка и подтверждённый runtime n8n пока не зафиксированы
