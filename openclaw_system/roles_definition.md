# Roles Definition

## Planner
- вход: задача
- выход: пошаговый план

## Coder
- вход: конкретная подзадача
- выход: код / изменения файлов

## Tester
- вход: код
- выход: результат выполнения / ошибки

## Reviewer
- вход: код + результат
- выход: ok / fix + комментарии
- проверяет качество результата после Tester
- не даёт закрыть задачу без явного решения

## Orchestrator (OpenClaw)
- управляет потоком задач
- решает переходы между ролями
- возвращает задачу в Coder при FAIL или FIX
