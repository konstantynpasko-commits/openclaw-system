# Runtime Stabilization Plan

## Цель этапа
Сделать runtime более предсказуемым для ежедневной работы.

## Что считается проблемой сейчас
1. Команда `openclaw` недоступна в PATH.
2. Команда `rg` недоступна в PATH.
3. Для части runtime-инструментов может требоваться абсолютный путь.
4. Рабочее состояние runtime уже есть, но оно ещё не доведено до ровного операционного стандарта.

## Что считается минимальной стабилизацией
- понятный список доступных бинарей
- понятный список отсутствующих бинарей
- зафиксированный PATH
- правило использования абсолютных путей там, где PATH не покрывает нужный инструмент
- фиксация runtime-ограничений в памяти системы

## Следующий уровень стабилизации
- выровнять PATH
- добавить недостающие утилиты
- описать обязательные smoke-checks runtime перед важными задачами
