# Refactor Template

## Вход
- что именно нужно улучшить
- почему текущая структура плоха
- какие ограничения нельзя нарушить

## Шаги
1. Зафиксировать цель refactor.
2. Определить, что должно остаться неизменным по поведению.
3. Выполнить структурные изменения.
4. Прогнать test.
5. Прогнать review.

## Критерий готовности
- структура улучшена
- поведение не сломано
- test = PASS
- review = OK
