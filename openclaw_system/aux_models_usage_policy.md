# Auxiliary Models Usage Policy

## Когда вспомогательная модель допустима
- когда нужен отдельный экспериментальный маршрут
- когда есть явная задача под дополнительный агент
- когда это не ломает основной предсказуемый execution path

## Когда не использовать
- как автоматический fallback без анализа
- как хаотичное ускорение за счёт параллельного шума
- как замену основного стабильного маршрута без подтверждения

## Политика осторожности
Вспомогательные модели — это расширение возможностей, а не повод усложнять систему без контроля.
