# Tester Checklist

## Назначение
Этот файл фиксирует минимальный контур проверки для Этапа 1.

## Вход
- изменения от Coder
- команда или способ проверки из плана Planner

## Обязательные действия Tester
1. Проверить, что изменения действительно записаны в нужные файлы.
2. Запустить минимальную доступную проверку:
   - запуск файла / команды
   - чтение вывода
   - фиксация ошибки или успешного результата
3. Зафиксировать итог в одном из двух состояний:
   - PASS
   - FAIL

## Формат результата Tester
- статус: PASS / FAIL
- команда проверки
- фактический результат
- что нужно исправить, если FAIL

## Жёсткое правило
- без результата Tester задача не может перейти дальше по workflow
