# Stage 2 Review Cycle

## Цель этапа
Сделать review обязательным системным слоем после тестирования.

## Полный цикл Этапа 2
1. Orchestrator принимает задачу.
2. Planner создаёт план.
3. Coder выполняет подзадачу.
4. Tester проверяет результат.
5. Если Tester = FAIL → возврат в Coder.
6. Если Tester = PASS → Reviewer делает проверку качества.
7. Если Reviewer = FIX → возврат в Coder.
8. После исправлений снова идут Tester и Reviewer.
9. Только последовательность PASS → OK позволяет считать задачу завершённой.

## Формула принятия
- FAIL = назад в Coder
- FIX = назад в Coder
- PASS + OK = задача может считаться выполненной
