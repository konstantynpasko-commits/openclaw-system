# Risk Tiers

## SAFE
Примеры:
- чтение файлов
- поиск по вебу
- чтение логов
- запись новых внутренних markdown/json файлов
- мелкие недеструктивные правки внутри workspace

## CAUTION
Примеры:
- изменение конфигов
- запуск shell-команд с побочным эффектом
- изменение существующих project files
- browser-действия на внешних сайтах
- создание cron-задач

## RESTRICTED
Примеры:
- удаление файлов
- restart/reconfigure системных компонентов
- действия, затрагивающие внешние сервисы
- правки, которые могут ломать runtime
- массовые изменения по проекту

## BLOCKED
Примеры:
- опасные действия без явного разрешения
- разрушительные команды без подтверждения
- действия с неясным последствием
- любые операции, которые выходят за рамки текущего запроса и контроля
