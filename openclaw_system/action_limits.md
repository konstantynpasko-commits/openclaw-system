# Action Limits

## Ограничения действий системы
1. Не выполнять опасные операции без явного разрешения.
2. Не делать внешние изменения, если пользователь этого не просил или это не вытекает прямо из задачи.
3. Не считать неподтверждённые интеграции рабочими.
4. Не обходить workflow ради скорости.
5. Не помечать задачу done без обязательных стадий.
6. Не использовать разрушительные команды как путь по умолчанию.

## Практический смысл
Система должна оставаться полезной, но не становиться источником неконтролируемого риска.
